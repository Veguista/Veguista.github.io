Wave Function Collapse Package

ROLES : UX Designer & Developer

W.f.c. maps

A Modular Unity Package for procedurally generating maps using the Wave Function Collapse algorithm.

Workflow and interface designed to prevent errors and reduce the complexity of use, while maximizing flexibility.

ENGINE : Unity

GitHub Repository

Core features

Custom interface
The tool's interface was designed with UX in mind, giving users full control:
  • Enable tiles and their connections with a single click.
  • Give each tile more or less weight to control their spawn probability.
  • Visualize active and inactive connections for each tile.
built-in backtracking
The Wave Function Collapse algorithm can get stuck in unsolvable positions. They are related to a map's configuration and cannot be prevented.

To tackle this problem, I implemented a Drop-Out Stack which recorded the steps a grid took to collapse:
  • Enabled the algorithm to backtrack when it reached a block.
  • Gave control to users over the number of steps that they wanted to track.
scalable workflow
The configuration of tiles and maps is stored as Scriptable Objects. This let's users:
  • Share tiles between different map configurations.
  • Automatically identify tile groups using the content's folder structure.
  • Swap tiles and map configurations with one click.
  • Easily duplicate map configurations and transfer them between projects.
error safe
I identified problematic ways in which users could misuse the tool. Afterwards, I altered its design to make those situations impossible:
  • Automatic disconnections when a connected tile is disabled.
  • Connections with disabled tiles are not possible.
  • Amount of code required to use the tool consist of only 3 function calls.
